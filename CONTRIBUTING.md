# Contributing to Awesome Telegram Bots

Thanks for your interest in contributing! This list is community-driven, and every contribution helps keep the Telegram bot ecosystem well-documented.

## Guidelines

### Adding a Resource

1. Fork this repo.
2. Create a branch: `git checkout -b add-resource-name`
3. Add your resource to the appropriate section in `README.md`.
4. Submit a pull request.

### What We're Looking For

- Libraries, SDKs, and frameworks for building Telegram bots.
- Open source bots that are well-maintained and useful.
- Tutorials and guides that are up to date (Bot API 7.0+).
- Tools for bot development, testing, and deployment.
- Mini App resources, templates, and examples.

### Quality Standards

- **No duplicates.** Search existing entries before submitting.
- **Working links.** All URLs must be live and accessible.
- **Maintained projects.** Libraries should have commits within the last 12 months, unless they are feature-complete and stable.
- **Descriptions matter.** Every entry needs a concise, helpful description.
- **English only.** All entries and descriptions should be in English.
- **No spam.** Your project is welcome if it's genuinely useful, but don't submit low-effort repos just for visibility.

### Formatting

Follow the existing format:

```markdown
- [Resource Name](https://link.to/resource) - One-line description ending with a period.
```

- Use title case for resource names.
- Descriptions should be one sentence, starting with a capital letter and ending with a period.
- Keep descriptions under 120 characters.
- Place new entries at the end of the relevant section.

### Removals

If a link is broken or a project is abandoned (no commits in 2+ years, no response to issues), open an issue or submit a PR to remove it.

## Code of Conduct

Be respectful. This is a community resource. Unhelpful, rude, or spammy contributions will be closed without merge.
